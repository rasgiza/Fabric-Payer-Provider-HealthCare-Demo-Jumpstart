# Lakeside Health System — Provider Credentialing Requirements

**Effective Date:** January 1, 2025  
**Medical Staff Office & Credentialing Department**  
**Regulatory Basis:** CMS CoP, Joint Commission, NCQA Credentialing Standards  
**Applicable:** All providers practicing at Lakeside facilities (FAC001-FAC008)

---

## 1. Overview

Provider credentialing ensures every clinician practicing at Lakeside Health System meets quality, training, and competency standards. Proper credentialing also prevents **Out of Network** and **Invalid Code** claim denials that occur when providers are not properly enrolled with payers.

### 1.1 Impact on Revenue Cycle
- Provider not credentialed with payer → **100% claim denial** (Out of Network)
- Provider NPI inactive or incorrect → **Invalid Code** denial
- Average time from hire to full credentialing: **90-120 days**
- Revenue loss from credentialing delays: **$15,000-$30,000 per provider per month**

## 2. Initial Credentialing Requirements

### 2.1 Required Documents
| Document | Source | Verification |
|----------|--------|-------------|
| Medical degree (MD/DO/NP/PA) | Training institution | Primary source verification |
| State license (Michigan, Ohio for FAC008) | LARA (MI), OSMB (OH) | Primary source, active status |
| DEA certificate | DEA registraion | Required for prescribing: all 22 formulary medications |
| Board certification | ABMS specialty board | Primary source, current status |
| NPI number | NPPES | Active and correct taxonomy |
| Malpractice insurance | Insurance carrier | Minimum $1M/$3M coverage |
| Work history (10 years) | Applicant + verification | Gap analysis required |
| CME documentation | Specialty-specific | Minimum hours met |
| Peer references (3) | Professional colleagues | Written or verbal |
| Background check | National Practitioner Data Bank | NPDB query |
| OIG/GSA exclusion check | Federal databases | No exclusions |
| Hospital privilege history | All prior hospitals | Primary source |

## 3. Payer Enrollment

### 3.1 Payer Enrollment Requirements

Every credentialed provider MUST be enrolled with each payer to bill and receive payment:

| Payer | Enrollment Method | Timeline | Special Requirements |
|-------|------------------|----------|---------------------|
| BCBS MI (PAY001) | Online portal (Availity) | 30-60 days | Michigan license required |
| Aetna (PAY002) | CAQH ProView | 30-60 days | Group enrollment required |
| United Healthcare (PAY003) | UHC Provider Portal | 45-90 days | Credentialing committee review |
| Cigna (PAY004) | CAQH ProView | 30-60 days | Panel may be closed for some specialties |
| Humana (PAY005) | Humana Provider Portal | 30-60 days | Standard process |
| Medicare (PAY006/007) | PECOS (CMS enrollment) | 45-120 days | PECOS ID required before billing |
| Medicaid MI (PAY008) | CHAMPS system | 60-120 days | Fingerprinting, site visit may be required |
| Priority Health (PAY009) | Direct application | 30-45 days | Regional Michigan plan |
| Tricare (PAY010) | HNFS portal | 45-60 days | Military/DoD requirements |
| Workers Comp (PAY011) | Per state WC board | Varies | Employer network verification |

### 3.2 Effective Date Management
- **Critical:** Provider CANNOT bill payer until effective date is set
- Services rendered before effective date → **automatic denial**
- Retroactive enrollment possible but rare and payer-specific
- Financial impact of 90-day enrollment gap: **~$45,000-$90,000 per provider** in unbillable services

### 3.3 CAQH ProView (Universal Application)
Most commercial payers accept CAQH ProView as the single credentialing data source:
- Provider completes one application
- Payers pull data directly
- **Must be re-attested every 120 days** or application becomes inactive
- Inactive CAQH = enrollment delays and potential network termination

## 4. Recredentialing (Every 2-3 Years)

### 4.1 Recredentialing Cycle
| Component | Frequency | Action |
|-----------|-----------|--------|
| Full recredentialing | Every 3 years (Joint Commission) / Every 2 years (NCQA) | Complete review |
| License verification | Annually | Confirm active MI/OH license |
| Board certification | Per certification cycle | Confirm current status |
| OIG/GSA exclusion check | Monthly | Automated screening |
| NPDB query | Every 2 years | At recredentialing |
| DEA verification | Every 3 years (or per schedule) | Active and correct address |
| CME compliance | Annually | Specialty-specific hours |
| Performance review | At recredentialing | Quality metrics, peer review |

### 4.2 Performance Metrics at Recredentialing
| Metric | Source | Threshold |
|--------|--------|-----------|
| Claim denial rate (provider-specific) | fact_claim | < 8% overall |
| Coding accuracy | Audit results | > 95% accuracy |
| Patient complaints | Quality department | < 3 substantiated per year |
| Malpractice claims | Malpractice carrier | Reviewed by credentials committee |
| Quality metrics (HEDIS) | Quality department | Meeting targets for applicable measures |
| Peer review outcomes | Medical staff | No adverse actions |

## 6. Provider Directory Management

### 6.1 Accuracy Requirements
- CMS requires provider directories updated within **30 days** of any change
- Inaccurate directories = patient access issues + Out of Network denials
- Must include: name, specialty, address, phone, accepting new patients, languages

### 6.2 Directory Updates Trigger Events
| Event | Action Required | Timeline |
|-------|----------------|----------|
| New provider credentialed | Add to all payer directories | Within 30 days of effective date |
| Provider leaves system | Remove from all directories | Within 30 days of termination |
| Address/location change | Update all directories | Within 30 days |
| Panel status change | Update accepting-new-patients status | Within 30 days |
| Specialty change | Update specialty listing | Within 30 days |
