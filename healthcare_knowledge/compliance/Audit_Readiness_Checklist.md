# Lakeside Health System — Audit Readiness Checklist

**Effective Date:** January 1, 2025  
**Compliance & Internal Audit Department**  
**Audit Types Covered:** CMS (Medicare), State (Medicaid), Commercial Payer, Internal, OIG  
**Applicable:** All departments, all facilities (FAC001-FAC008)

---

## 1. Overview

Lakeside Health System must maintain continuous audit readiness across clinical, financial, and operational domains. Audits can be triggered by routine schedules, complaint investigations, statistical outliers in claims data, or risk-based selection.

### 1.1 Audit Frequency
| Audit Type | Frequency | Trigger |
|-----------|-----------|---------|
| CMS CERT (Comprehensive Error Rate Testing) | Annual sampling | Random Medicare claim sample |
| RAC (Recovery Audit Contractor) | Ongoing | Automated claim review for overpayments |
| OIG (Office of Inspector General) | Risk-based | Work plan focus areas (e.g., high-cost DRGs) |
| State Medicaid audit | Annual | Michigan DHHS program integrity |
| Commercial payer audit | Per contract | BCBS, Aetna, UHC clinical audits |
| Internal compliance audit | Quarterly | Proactive risk management |
| Joint Commission survey | Every 3 years | Accreditation renewal |
| HIPAA security audit | Annual | OCR risk assessment requirement |

### 1.2 Financial Risk from Audit Findings
| Finding Type | Financial Impact |
|-------------|----------------|
| Overpayment identification | Refund required within 60 days + interest |
| Extrapolated overpayment | Single findings multiplied across claim universe |
| Coding errors (systematic) | Corporate Integrity Agreement potential |
| Documentation deficiencies | Claim-by-claim denial + education mandate |
| False Claims Act violation | Triple damages + $13,946-$27,894 per claim (2024 inflation-adjusted) |
| Exclusion from programs | Loss of Medicare/Medicaid participation |

## 2. Clinical Documentation Audit Checklist

### 2.1 Inpatient Records (99221-99223)
For every sampled inpatient admission, verify:

- [ ] Admission order present and signed by attending physician
- [ ] H&P completed within 24 hours of admission
- [ ] H&P contains: chief complaint, HPI, ROS, exam, assessment, plan
- [ ] Admitting diagnosis documented with ICD-10 specificity (e.g., I50.9 not I50)
- [ ] All secondary diagnoses documented (comorbidities: I10, E11.9, E78.5, etc.)
- [ ] Daily progress notes present for each day of stay
- [ ] Consultations documented within 24 hours of request
- [ ] Operative reports completed within 24 hours (if surgery: 27447, 27130, 33533, 43239, 45378)
- [ ] Discharge summary completed within 48 hours
- [ ] Discharge summary includes: diagnoses, procedures, hospital course, discharge meds, follow-up
- [ ] Medication reconciliation at admission and discharge
- [ ] Informed consent present for all procedures
- [ ] Code status documented
- [ ] Patient identification verified (two identifiers)

### 2.2 Outpatient Records (99213-99215)
For every sampled outpatient visit, verify:

- [ ] Visit note present and signed by provider
- [ ] Chief complaint documented
- [ ] History of present illness adequate for visit level
- [ ] Assessment contains diagnosis with ICD-10 specificity
- [ ] Plan addresses each diagnosis
- [ ] Medical decision making complexity supports E&M level billed
- [ ] Time-based billing (if used) has total time documented
- [ ] Controlled substance prescriptions include PDMP check documentation
- [ ] Any ordered tests have clinical indication documented

### 2.3 ED Records (99283-99285)
For every sampled ED visit, verify:

- [ ] Triage assessment documented with time
- [ ] Physician evaluation documented with time
- [ ] History and medical decision-making support ED level billed
- [ ] All diagnostic results reviewed and documented (labs 85025/80053, imaging 71046/74177, ECG 93000)
- [ ] Disposition documented with rationale
- [ ] If admitted: admission criteria documented
- [ ] If discharged: discharge instructions, follow-up plan, return precautions
- [ ] If transferred: EMTALA compliance documented

## 3. Coding Audit Checklist

### 3.1 ICD-10 Coding Accuracy
- [ ] Principal diagnosis = condition chiefly responsible for the encounter
- [ ] All secondary diagnoses captured (chronic conditions actively managed)
- [ ] Codes are specific to highest level (no unspecified when specificity available)
- [ ] No assumption coding — every code has documentation support
- [ ] Gender-specific codes match patient gender (e.g., C50.911 for female only)
- [ ] Excludes notes respected (codes that cannot be used together)
- [ ] Sequencing follows Official Coding Guidelines

#### Common ICD-10 Codes to Audit at Lakeside
| Code | Verify |
|------|--------|
| I50.9 | CHF type specified (systolic/diastolic/combined) if documented |
| E11.9 | Complications documented? May need E11.2x, E11.4x, etc. |
| J44.9 | Acute exacerbation? Should be J44.1 if applicable |
| A41.9 | Organism specified? Consider more specific sepsis code |
| J18.9 | Organism identified? Consider J13, J15.x if culture results available |
| I10 | Hypertensive crisis? Emergency/urgency codes different |
| R10.9 | Should be specified to location (R10.1, R10.3, etc.) |

### 3.2 CPT Coding Accuracy
- [ ] CPT code matches the procedure/service documented
- [ ] Appropriate modifiers applied
- [ ] No unbundling (NCCI edit compliance)
- [ ] Units of service correct
- [ ] Global period respected for surgical follow-ups
- [ ] Evaluation and management level supported by documentation

#### High-Risk CPT Codes for Audit
| CPT | Risk Area |
|-----|-----------|
| 99215 | High-level E&M — documentation must support high complexity MDM |
| 99285 | Highest ED level — ensure documentation supports severity |
| 99223 | Initial hospital care, high — complex MDM required |
| 74177 | CT with contrast — medical necessity for each order |
| 93306 | Echocardiogram — frequency and indication documented |
| 27447/27130 | Joint replacement — full operative report, implant documentation |
| 33533 | CABG — operative details, indications, cath report |

### 3.3 Modifier Audit
- [ ] Modifier 25 (separate E&M): Clear documentation of separately identifiable service
- [ ] Modifier 59 (distinct procedure): Different anatomic site or separate encounter documented
- [ ] Modifier 50 (bilateral): Both sides documented in operative report
- [ ] Modifier 76/77 (repeat procedure): Clinical need for repeat documented
- [ ] Modifier 26 (professional component): Physician interpretation documented

## 4. Billing and Claims Audit Checklist

### 4.1 Claim Accuracy
- [ ] Patient demographics match insurance records
- [ ] Insurance/payer information verified and current
- [ ] Date of service matches clinical documentation
- [ ] Referring/ordering provider NPI correct and active
- [ ] Rendering provider NPI correct, enrolled with payer
- [ ] Facility NPI matches service location
- [ ] Place of service code correct
- [ ] Prior authorization number present (if required)
- [ ] Charge amount matches fee schedule
- [ ] No duplicate claims in system

### 4.2 Prior Authorization Audit
- [ ] PA obtained before service (for PA-required services)
- [ ] PA authorization number on claim
- [ ] PA covers specific CPT code billed
- [ ] PA date range covers date of service
- [ ] PA not expired at time of claim submission
- [ ] Emergency retro PA submitted within 48 hours (if applicable)

### 4.3 Denial Management Audit
- [ ] All denials reviewed within 5 business days
- [ ] Actionable denials appealed within 30 days
- [ ] Appeal documentation complete and compelling
- [ ] Appeal success tracked by denial reason
- [ ] Systemic denial patterns identified and escalated
- [ ] Denial root cause analysis performed monthly

## 5. Quality and Compliance Audit Checklist

### 5.1 HIPAA Compliance
- [ ] Annual HIPAA training completed by all workforce members
- [ ] Authorization forms obtained for non-TPO disclosures
- [ ] Notice of Privacy Practices provided to patients
- [ ] Minimum necessary standard applied for all access
- [ ] Business Associate Agreements current for all vendors
- [ ] Breach incident log up to date
- [ ] Security risk assessment completed within 12 months
- [ ] Access audit logs reviewed quarterly

### 5.2 Quality Measures (HEDIS/Stars)
- [ ] HEDIS measure performance tracked monthly
- [ ] Gap lists generated and distributed to care teams
- [ ] Medication adherence (PDC) monitored monthly
- [ ] Readmission cases reviewed within 48 hours
- [ ] Quality improvement actions documented
- [ ] Annual quality reporting submitted on time

### 5.3 CMS Condition of Participation (CoP) Compliance
- [ ] Patients' rights prominently posted
- [ ] Grievance process accessible
- [ ] Infection control program active
- [ ] Medical staff bylaws current
- [ ] Emergency preparedness plan tested annually
- [ ] QAPI (Quality Assessment and Performance Improvement) program active

## 6. Audit Response Protocol

### 6.1 When Notification Received
| Step | Action | Timeline | Owner |
|------|--------|---------|-------|
| 1 | Notify Compliance Officer | Within 24 hours | Department receiving notification |
| 2 | Assemble response team | Within 48 hours | Compliance Officer |
| 3 | Document preservation notice | Immediately | Legal + IT |
| 4 | Gather requested records | Per auditor timeline | HIM + Billing |
| 5 | Legal review of sample | Before submission | Legal counsel |
| 6 | Submit response | Per deadline | Compliance Officer |
| 7 | Track findings and corrective actions | Ongoing | Compliance + Quality |

### 6.2 Response Team by Audit Type
| Audit Type | Core Team |
|-----------|-----------|
| CMS/RAC | Compliance Officer, Coding Manager, Revenue Cycle Director, Legal |
| Medicaid | Compliance Officer, Government Programs Director, Legal |
| Commercial Payer | Managed Care Director, Revenue Cycle, relevant department |
| OIG | Compliance Officer, Legal Counsel, CFO, department leadership |
| Joint Commission | Quality Director, CNO, CMO, department managers |
| Internal | Internal Audit, Compliance, relevant department |

### 6.3 Corrective Action Plan Template
| Element | Content |
|---------|--------|
| Finding | Specific audit finding |
| Root cause | Why it occurred (reference denial RCA framework) |
| Corrective action | Specific steps to fix |
| Responsible party | Name and title |
| Target date | Completion deadline |
| Monitoring plan | How to verify correction is sustained |
| Evidence of completion | Documentation of fix |

### 7.2 Analytics-Driven Audit Targets
Use the Data Agent / Gold analytics to identify audit focus areas:
- "Which diagnosis codes have the highest denial rate?" → Coding audit focus
- "Which providers have denial rates above 10%?" → Provider-specific documentation audit
- "Which payer denies the most claims for Missing Documentation?" → Process audit
- "What is the trend in Prior Auth Required denials?" → PA process audit
- "Which facilities have the highest readmission rate?" → Clinical care audit
